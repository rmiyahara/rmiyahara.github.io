Do Not Disturb

August 2018 - Present

Technology Used: ReactNative, Android Studio, Java, HTML

As a student at UCLA where nearly everyone has a roommate or two, I noticed a major problem when it came to roommate communication. Often times, issues would stem from having overnight guests over. Communicating this can be uncomfortable for some, but when it isn't communicated at all, awkward walk-ins are bound to happen. The goal of this application is to provide roommates with a discrete way to tell their roommates when the room is occupied and when they are free to come back in.

This is implemented by using a simple homescreen highlighting one button which starts a timer and sends a push notification to connected roommates. The timer length is agreed upon by all roommates beforehand and will notify roommates with another push notification when the timer reaches zero. Roommates can easily be added in the "Roommates" tab on the left and various settings, such as timer length, can be toggled in the "Settings" tab.

Development for this application is currently ongoing with an expected completion date by the end of the year.

No Duines

February 2018 - March 2018

Technology Used: Arduino Software, C, Basic circuit reworking

Despite being more than 15 years old, Nintendo's Super Smash Bros. Melee still thrives as a game with an active fanbase. Melee had the highest viewer count during Evolution 2018, the most prestigious fighting game tournament of the year outliving all other fighting games matching its age. One of its defining factors would be its demanding techniques, something the community dubbed "Tech Skill".

Each of these techniques has a unique name such as Wavedashing, L-Cancelling, etc. While some of these are regularly used during competitive play, some of these techniques have an execution barrier that is too high for humans. These are discovered using tool assisted speedruns (TAS) where the game can be played out frame by frame. While the execution may be too high for humans, an Arduino has no such limits.

By soldering an Arduino to the motherboard of a controller, I am able to have my character perform these high-precision techniques at the push of a button. Examples of these can be seen in the video above. This modification is not meant for tournament play as it provides clear, unfair advantages. While this project is finished, I occasionally come back and add a few new macros.

Ice Man

July 2017 - August 2017

Technology Used: Visual Studio, C++

This was my final project for UCLA's Computer Science 32: Introduction to Computer Science II. The objective of the game is to move the Ice Man throughout the field looking for oil barrels. He will quickly dig away any ice in his path and collect various power ups as he makes contact with them. The Ice Man is not alone however; following him are protestors who are looking to end Ice Man's quest. They will eat away at Ice Man's health until he is forced to leave. In this case, the player will lost a life and start the level over. All power ups, health, and score is kept on the top of the screen.

This project's main obstacle was creating a useful inheritence heirarchy for the player, enemies, and power ups. I wanted to avoid rewriting code, but had to make sure everything was working properly. Once I had finilized the heirarchy, the next big issue was writing the behavior for the protestors. While some of them move randomly in straight lines, others will move towards the player if they are within a certain amount of steps away from the player (the number of steps increases with the level). On top of that, if a protestor is defeated, they must then leave the field, taking the shortest path to their spawn point. I eventually accomplished did this using an queue-based algorithm on the cleared part of the field. Overall, this project really challeneged my problem solving skills as well as my familiarity with C++.

Health Tracker

November 2017

Technology Used: Android Studio, React Native, SciKit HTML, JavaScript, Python

This project was made for UCLA's Hack on the Hill. This hackathon lasted for 16 hours where teams began working on various projects. This one was my first hackathon and really opened to my eyes to taking coding beyond the classroom. This project's main objective was to use the Hill's extensive dining menu and the user's eating habits to construct a suggested diet. As the user continues to log their foods, the application could continuously suggest meals which would be healthier and more enjoyable to the user.

The biggest issue we ran into during our development was time. Since development was only a day long, we had to cut a few features as well as excluding certain dining halls. On top of this, we were all working with technology we had little to no experience with at the time which made everything take longer than expected. While I was working on the front-end, we eventually all had to input nutritional values manually due to the values being saved in a picture on the dining menu site. Without the time to figure out image recognition, each item on the menu had to be added by hand making this process quite tedious.

In the end, our team was able to present the application with limited functionality. While very stressful, the entire hackathon experience was enlightening and helped spark my love for coding.

Birthday Tracker

January 2018 - February 2018

Technology Used: Python, Windows Batch Scripting

This project is a short Python script which includes a list of birthdays and corresponding names. When the script is run, it notifies the user which birthday is either today or coming soon. The idea for this program came to me while I was learning Python. During this time, I held a leadership position for UCLA's Kyodo Taiko, a Taiko performance group. One of the responsibilities of this position was to acknowledge each member's birthday. At the time, I didn't really have a system to do this. I would check the roster everyday which became very tedious over the course of the year. I used this as an oppertunity to apply Python to my everyday life.

While this solution worked, I still had to remember to run it everyday. Luckily for me, I had just finished my unit in Linux Bash Scripting in class. I applied what I had learned with a few changes since I was working with Windows this time around to get the script to run each time I booted my computer. After I got this to work, I could finally stop worrying about missing a birthday for the rest of my term.
